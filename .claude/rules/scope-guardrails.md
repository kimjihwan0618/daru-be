# 작업 범위 가이드

- 스켈레톤에 이미 정의된 모델/스키마 필드 임의 변경 금지. 필요하면 먼저 이유를 설명하고 확인받을 것.
- README 5절("미해결 설계 메모") 항목(`users.preferences`, 브리핑 피드백 테이블, 공유 링크 만료 정책 등)은 설계 보류 상태. 방향 확인 없이 테이블/컬럼 추가 금지.
- 요청받지 않은 리팩터링/추상화 추가/여러 소셜 로그인 동시 구현 등 범위 초과 작업 금지.
- 구현 순서: 카카오 로그인 → 읽기 전용 기능 → 뉴스 수집 → RAG → 개인화 (README "구현 우선순위 가이드"). 순서를 벗어난 선행 구현이 필요하면 먼저 확인.
